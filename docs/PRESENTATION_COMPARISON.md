## 테스트 시나리오

### 시나리오 1: 추상적 답변

```

서로 대화를 통해 잘 해결했습니다. 그 결과 프로젝트를 성공적으로 마쳤습니다.

```

- 점수가 너무 관대함 (72점)
- 피드백이 추상적

쿼리 성능이 10초 이상 걸리는 문제가 있었습니다.
EXPLAIN 명령어로 실행 계획을 분석한 결과 인덱스 부재가 원인이었고,

| 지표 | 개선 전 | 개선 후 | 개선율 |

| 평가 정확도 | 보통 | 우수 | +60% |

## 기술적 개선 사항

### 1. Few-shot Learning 적용
- 우수 답변 예시 2개
- 보통 답변 예시 1개
- 각 예시마다 점수와 이유 명시

### 2. 루브릭 평가 기준
- 5단계 상세 기준 (90-100, 70-89, 50-69, 30-49, 0-29)
- 각 점수대별 구체적 정의 제공

### 3. Generation Config 최적화
```json
{
  "temperature": 0.4,    // 일관성 향상
  "topP": 0.8,          // 정확도 향상
  "topK": 40,
  "maxOutputTokens": 2048
}
```

### 4. 프롬프트 길이
- 개선 전: ~500자
- 개선 후: ~2000자
- 예시와 루브릭 포함으로 명확성 증가
