# 피드백 품질 수동 평가 가이드

전문성이나 프로그래밍 없이 피드백 품질을 평가하는 방법입니다.

## 📋 평가 체크리스트

### 1. 구체성 체크
- [ ] 숫자가 있는가? (예: "3개", "2일", "50%")
- [ ] 고유명사가 있는가? (예: "RESTful", "STAR 기법")
- [ ] "좋다", "부족하다" 같은 막연한 표현이 적은가?

**점수:**
- 3개 모두 체크: 90-100점
- 2개 체크: 70-89점
- 1개 체크: 50-69점
- 0개 체크: 0-49점

---

### 2. 실행 가능성 체크
- [ ] "~를 추가하세요" 같은 명확한 액션이 있는가?
- [ ] 예시가 제공되는가?
- [ ] "더 잘하세요" 같은 추상적 조언이 아닌가?

**점수:**
- 3개 모두 체크: 90-100점
- 2개 체크: 70-89점
- 1개 체크: 50-69점
- 0개 체크: 0-49점

---

### 3. 균형성 체크
- [ ] 강점이 2개 이상 언급되는가?
- [ ] 약점이 2개 이상 언급되는가?
- [ ] 강점/약점이 균형있게 제시되는가?

**점수:**
- 3개 모두 체크: 90-100점
- 2개 체크: 70-89점
- 1개 체크: 50-69점
- 0개 체크: 0-49점

---

## 📊 샘플 평가 예시

### 예시 1: 나쁜 피드백
```
"전반적으로 좋은 답변입니다. 조금 더 구체적으로 말하면 좋겠습니다."
```

**평가:**
- 구체성: ❌❌❌ (0점) - 숫자도, 고유명사도 없음
- 실행가능성: ❌❌❌ (0점) - "조금 더"가 뭔지 모호
- 균형성: ❌❌✅ (30점) - 강점만 언급

**종합: 10점**

---

### 예시 2: 좋은 피드백
```
강점:
1. STAR 구조(상황-과제-행동-결과)가 명확합니다
2. "2일간", "5명" 등 수치가 3개 포함되어 구체적입니다

약점:
1. 결과 부분에 정량적 성과가 부족합니다 (예: "A학점", "성능 30% 개선" 추가 권장)
2. 팀원 반응이나 후속 영향이 생략되었습니다
```

**평가:**
- 구체성: ✅✅✅ (95점) - STAR, 수치, 예시 모두 있음
- 실행가능성: ✅✅✅ (95점) - "정량적 성과 추가", 구체적 예시 제공
- 균형성: ✅✅✅ (100점) - 강점 2개, 약점 2개 균형

**종합: 97점**

---

## 🎯 빠른 평가 방법

5분 안에 평가하는 방법:

1. **Ctrl+F로 검색**
   - "점", "개", "명", "일" → 수치 있는지 확인
   - "추가하세요", "권장" → 실행 가능한 조언 있는지 확인

2. **문장 개수 세기**
   - 강점 문장: ___개
   - 약점 문장: ___개
   - 균형 있으면 OK

3. **총 종합 점수**
   ```
   (구체성 + 실행가능성 + 균형성) / 3 = 종합 점수
   ```

## 💡 팁

1. **비교는 최소 3개 샘플로**
   - 추상적 답변 1개
   - 보통 답변 1개
   - 구체적 답변 1개

2. **같은 답변으로 평가**
   - 변수를 통제해야 비교 가능

3. **10개 평가하면 통계적으로 유의미**
   - 3-5개: 대략적 경향
   - 10개 이상: 신뢰 가능한 데이터
